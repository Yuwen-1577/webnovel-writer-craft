## 一、叙事视角操作规范

### 默认视角：第三人称限制

绝大多数网文首选。好处：灵活+亲密感，章节间可切换视角人物。

### 开篇锁定规则

- **前 30 章**：严格锁定主角单一视角。读者还没建立代入感，任何视角切换都是干扰。
- **中期展开**：关键节点（反派布局、盟友暗线）可用独立章节呈现非主角视角，每章开头明确标注。
- **大高潮**：多视角交叉剪辑（平行蒙太奇），多条线同时推进到交汇点。

### POV 切换检查清单

- [ ] 一个场景 = 一个视角，不在同一场景内跳切
- [ ] 切换有信息收益（新视角能看到主角看不到的信息）
- [ ] 视角人物不超过 3-4 人
- [ ] 不滑进配角内心（"主角能看到这个人的表情吗？能读到他的心声吗？"）

## 二、白描与渲染的比例控制

### 核心原则：白描为主，渲染为辅（7:3 到 8:2）

**白描**（速写式，不加修饰）：打斗动作、对话推进、情节转折、保持阅读速度。

**渲染**（铺陈细节）：情感高潮、重大场景沉浸感、世界观展示、角色心理转折。

### 过度渲染的危险信号

- 大段心理独白
- 连续三个以上比喻
- 超过 150 字的环境描写
- → 读者会跳读

## 三、对话技法

### 海明威冰山对话法

操作步骤：
1. 写对话前，先在脑中写出每个角色**真正想说的**和**实际说出的**
2. 删除所有"解释性台词"
3. 让角色用非语言行为暗示情绪（攥紧杯子、反复看手机、说话越来越快）
4. 信任读者

### 潜台词技巧

| 技巧 | 示例 |
|------|------|
| 回避 | A: "你昨晚去哪了" B: "今天的天气不错" |
| 反问 | "你觉得呢？" = "我不想告诉你" |
| 行动打断 | 角色突然站起来倒水——身体语言比嘴巴诚实 |
| 过度解释 | 越在撒谎的角色，越会说太多 |

### 网文对话规则

- 每段对话不超过 4-6 个来回
- 对话标签少用，用动作代替："走吧。"他拉开门。
- 方言口癖克制使用（一两个标志性口癖即可）
- 内心独白不应长过对话本身

## 四、节奏控制：SCE 框架

Dwight Swain 的场景-续接框架，特别适合网文单元化叙事：

```
Scene（场景）
  角色有明确目标 → 遇到冲突 → 以"灾难"或悬念结束
    ↓
Sequel（续接）
  情绪反应 → 理性分析 → 做出新决策
    ↓
下一个 Scene
  基于新决策开始新目标 → 循环
```

每 2-3 章完成一个 SCE 循环，每 10-20 章完成一个大循环。

### 场景 vs 概述

- 场景（实时展开）：70-80%，用于冲突/决策/爽点/情感进展
- 概述（压缩叙述）：20-30%，用于旅途/等待/重复训练

### 时间扩展（慢镜头）

- 打斗动作拆成 5 个分解步骤
- 决策瞬间展开大量内心分析
- "子弹时间"：主角出手瞬间，切换到旁观者震惊反应

## 五、章末钩子设计

每章必须有钩子。五种类型交替使用：

| 类型 | 示例 | 适用题材 |
|------|------|---------|
| **悬念型** | "门推开了，站在门口的人让他彻底愣住。" | 通用 |
| **反转型** | "他以为自己赢了。直到他看见角落里那面镜子。" | 悬疑/都市 |
| **决策型** | "左边是密道，右边是大门。他深吸一口气——" | 通用 |
| **情感型** | "她转身走进雨里。他张了张嘴，一个字也没说出来。" | 言情 |
| **世界观型** | "石碑上那行字，和他梦里见过的一模一样。" | 玄幻/无限流 |

**操作原则**：
- 章末最后一段不超过 3 行
- 交替使用不同类型（同一种连续 3 次以上会免疫）
- 不要每次都用悬念型——情感型和世界观型是很好的调剂

## 六、感官描写优先级

**触觉 > 嗅觉 > 听觉 > 视觉 > 味觉**

- 触觉最私密，天然制造亲密感和在场感
- 嗅觉直接连接记忆和情感（普鲁斯特效应）
- 视觉容易过度使用，要节制
- 感官细节必须有信息量（"空气中有铁锈味"暗示血）

### "展示而非讲述"的网文化

不只是展示角色行为，还要通过**他人的震惊反应**放大爽感：

讲述："他很强。"
文学展示："他的拳头攥紧了，指节发白。"
**网文展示**："全场鸦雀无声。刚才还在嘲笑他的人，此刻双腿发软，'扑通'跪在地上。'前……前辈……'"

## 七、信息投放节奏

### 三种投放方式

| 方式 | 操作 | 频率 |
|------|------|------|
| **滴灌式** | 每章只投放 1-2 个新设定 | 最常用 |
| **需求式** | 角色遇到新情况时顺势解释 | 按需 |
| **炸弹式** | 重大转折点投放震撼性真相 | 偶尔 |

### 避免 Info-dump

- 连续三段以上纯设定介绍 = 危险信号
- "众所周知"开头 = 必删
- 力量体系一次性全部列出 = 必删
- 替代方案：嵌入行动中、用冲突揭示、延迟满足

### 冰山操作法

作者知道 100%，读者感知 10-15%。剩余通过角色反应、文化细节、偶尔暗示传递，永远不直接说出来。

为每个主要角色写"冰山档案"：公开面（读者看到的）+ 隐秘面（作者知道的）。隐秘面不需要全部揭开，但它影响角色的每一个决定。
