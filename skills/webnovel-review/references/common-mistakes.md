## 快速自检清单

| 检查项 | 通过标准 | 违规后果 |
|--------|----------|----------|
| 战力合理 | 主角实力 ≤ state.json | 破坏战力体系 |
| 爽点存在 | 至少 1 个爽点/章 | 读者流失 |
| 实体识别 | 新实体可被 Data Agent 自动提取 | 设定混乱 |
| 人物一致 | 对话/行为符合人设 | OOC 出戏 |
| 节奏平衡 | 未连续 5+ 章同一 Strand | 审美疲劳 |
| 伏笔追踪 | 伏笔可被识别并记录 | 遗忘伏笔 |
| **价值转变** | 场景结束时核心价值状态与开始时不同 | 流水账/水字数 |
| **冲突有效性** | 冲突产生"期望鸿沟"（读者预期被打破） | 平淡无张力 |
| **角色弧光一致** | 主角行为与选定弧光类型一致 | 弧光断裂 |
| **伏笔张力** | 活跃伏笔在制造悬念而非只是"埋了" | 伏笔无感 |
| **风格一致性** | 本章风格与 state.json writing_style 一致（或切换有理由） | 风格漂移 |

## 叙事质量评分系统

每章审查时，对以下 5 个维度打分（1-10），生成结构化评分 JSON：

| 维度 | 评分标准 | 1-3（差） | 4-6（中） | 7-10（好） |
|------|---------|----------|----------|-----------|
| **价值转变** | 场景开始→结束是否有可识别的价值状态变化 | 开始和结束状态相同 | 有变化但不够清晰 | 明确的价值翻转，读者能感受到 |
| **冲突有效性** | 冲突是否产生"期望鸿沟" | 读者能猜到结果 | 有一定意外但不够震撼 | 结果出乎意料但合理 |
| **角色弧光** | 主角行为是否与选定弧光类型一致 | 行为与弧光矛盾 | 基本一致但缺少推进 | 弧光清晰推进，读者能感受到成长/堕落 |
| **风格一致性** | 文字风格是否与选定流派匹配 | 风格混乱，不像任何流派 | 基本匹配但有偏差 | 风格精准，可辨识度高 |
| **伏笔张力** | 活跃伏笔是否在制造悬念 | 伏笔存在但读者无感 | 有悬念感但不够强 | 读者迫切想知道答案 |

**输出格式**（保存到 `.webnovel/tmp/review_narrative_scores.json`）：

```json
{
  "chapter": 42,
  "scores": {
    "value_change": 7,
    "conflict_effectiveness": 6,
    "arc_consistency": 8,
    "style_consistency": 5,
    "foreshadow_tension": 7
  },
  "total": 33,
  "max": 50,
  "grade": "B+",
  "issues": ["风格偏移：本章对话风格从'文青叙事流'偏向'白描爽文流'"],
  "strengths": ["角色弧光推进清晰：主角首次选择牺牲而非自保"]
}
```

**等级标准**：
| 总分 | 等级 | 判定 |
|------|------|------|
| 45-50 | A | 优秀，可直接发布 |
| 38-44 | A- | 良好，微调即可 |
| 33-37 | B+ | 合格，有改进空间 |
| 25-32 | B | 及格，需要修改 |
| 18-24 | C | 不及格，需要重写部分段落 |
| <18 | D | 严重问题，需要重写 |

**趋势追踪**：每次评分追加到 `.webnovel/metrics/narrative_scores.jsonl`，可查询：
- 最近 10 章平均分
- 各维度趋势（哪个维度在下降）
- 风格切换前后的分数对比

<errors>
❌ 战力崩坏（越级秒杀无理由） → ✅ 检查 state.json，合理设定对手
❌ 爽点缺失（整章无冲突） → ✅ 至少 1 个爽点，过渡章也要有
❌ 擅自发明（新实体描写模糊） → ✅ 确保新实体有明确名称和描写
❌ 人物 OOC（性格突变无铺垫） → ✅ 参考角色卡，有转变需铺垫
❌ 节奏单调（连续 10 章纯战斗） → ✅ 检查 strand_tracker，切换 Strand
❌ 伏笔遗忘（超 100 章未回收） → ✅ 核心伏笔 50 章内回收
❌ 对话脸谱化（反派只会喊口号） → ✅ B级以上反派要有智商
❌ 价值无转变（场景开始和结束状态一样） → ✅ 每场景至少一个维度的价值状态变化（详见 scene-value-change.md）
❌ 冲突无张力（读者能猜到结果） → ✅ 制造 McKee 式"期望鸿沟"
❌ 弧光断裂（主角行为与弧光类型矛盾） → ✅ 正向弧主角不应在关键时刻无理由退缩
❌ 伏笔无感（埋了但读者不紧张） → ✅ 用悬念（读者知道有危险）替代惊奇（突然爆炸）
</errors>
